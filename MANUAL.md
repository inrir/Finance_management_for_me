## 1. 개요

**개인 금융 자산 관리** 앱은 주식·ETF·가상화폐·예금·현금·부동산 등 다양한 자산을 한 곳에서 수기 입력 방식으로 관리하는 웹 애플리케이션입니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| 수기 입력 기반 | 실시간 시세 연동 없음. 현재가를 직접 입력하여 수익률 계산 |
| 로컬 저장 | 모든 데이터는 브라우저 IndexedDB에 저장. 서버 전송 없음 |
| 다중 통화 | KRW · USD · JPY 지원. 외화 자산은 원화 환산액을 함께 표시 |
| 무료 운영 | 별도 서버·클라우드 비용 없음 |

### 지원 자산 유형

| 유형 | 설명 | 입력 방식 |
|------|------|-----------|
| 주식 | 국내·해외 개별 종목 | 현재가 + 수량 |
| ETF | 상장지수펀드 | 현재가 + 수량 |
| 가상화폐 | 비트코인 등 디지털 자산 | 현재가 + 수량 |
| 기타 | 분류되지 않는 투자 자산 | 현재가 + 수량 |
| 부동산 | 아파트, 토지 등 | 현재 시세 (수량=1 고정) |
| 예·적금 | 정기예금, 적금 | 보유금액 (단가=1 고정) |
| 현금 | 지갑, 통장 잔액 | 보유금액 (단가=1 고정) |

## 2. 시작하기

### 접속

브라우저에서 앱 URL에 접속합니다. (로컬 개발: `http://localhost:5173`)

### 초기 설정 (권장)

1. **설정 > 환율 설정**으로 이동합니다.
2. USD, JPY의 현재 환율을 입력합니다.
   - 예: USD `1,350`, JPY `9.0` (1 외화 = ? KRW 형식)
3. 환율은 자동 조회되지 않으므로 주기적으로 수동 업데이트해야 합니다.

## 3. 자산 관리

### 3-1. 자산 추가

**경로**: 사이드바 > 자산 관리 > `+ 자산 추가` 버튼

입력 항목:

| 항목 | 필수 | 설명 |
|------|------|------|
| 자산명 | ✅ | 표시할 이름 (예: 삼성전자, S&P500 ETF) |
| 자산 유형 | ✅ | 주식·ETF·가상화폐·기타·부동산·예·적금·현금 |
| 시장 | ✅ | KRX·NYSE·NASDAQ·CRYPTO·해당없음 |
| 통화 | ✅ | KRW·USD·JPY |
| 종목코드 | - | 티커 (예: 005930, SPY, BTC) |
| **금액 입력** (유형별 상이) | ✅ | 아래 참조 |
| 메모 | - | 자유 입력 |

**유형별 금액 입력**

- **주식·ETF·가상화폐·기타**: "현재가"와 "수량" 두 항목 입력
- **예·적금·현금**: "보유금액" 하나만 입력 (현재가=1 자동 설정)
- **부동산**: "현재 시세" 하나만 입력 (수량=1 자동 설정)

### 3-2. 자산 목록 확인

**경로**: 사이드바 > 자산 관리

테이블에서 각 자산의 현재가·평가금액·수익률·평가손익을 확인할 수 있습니다.

**외화 자산 (USD/JPY)**의 경우 평가금액·평가손익 아래에 `≈ ₩xxx` 형식으로 원화 환산액이 함께 표시됩니다.

### 3-3. 현재가 업데이트

현재가는 자동으로 갱신되지 않으므로 수동으로 업데이트해야 합니다.

**개별 업데이트**
- 자산 목록의 "현재가" 값을 클릭하면 업데이트 모달이 열립니다.
- 또는 자산 상세 페이지 > "현재가 업데이트" 버튼을 사용합니다.

**일괄 업데이트**
1. 자산 목록 우측 상단 **"현재가 일괄 입력"** 버튼 클릭
2. 주식·ETF·가상화폐·부동산 행에 인라인 입력 필드가 나타납니다.
3. 변경할 현재가를 입력 후 **"저장"** 버튼 클릭
4. 예·적금·현금은 현재가 입력 대상이 아니므로 나타나지 않습니다.

> **참고**: 예·적금·현금의 잔액 변동은 거래 내역(DEPOSIT/WITHDRAW)으로 기록합니다.

### 3-4. 자산 편집

**경로**: 자산 상세 페이지 > "편집" 버튼

자산명·유형·시장·통화·종목코드·메모·보유금액(유형에 따라)을 수정할 수 있습니다.

### 3-5. 자산 삭제

**경로**: 자산 목록 우측 "삭제" 버튼 또는 자산 상세 > "삭제" 버튼

> ⚠️ 자산 삭제 시 해당 자산의 **모든 거래 내역도 함께 삭제**됩니다. 복구되지 않으므로 주의하세요.

### 3-6. 자산 상세 페이지

**경로**: 자산 목록 > 자산명 클릭

자산 유형에 따라 다른 통계 카드가 표시됩니다.

| 유형 | 표시 카드 |
|------|-----------|
| 주식·ETF·가상화폐·기타 | 현재 평가금액 / 평균 매입단가 / 평가손익 / 보유수량 |
| 부동산 | 현재 시세 / 취득가액 / 평가손익 |
| 예·적금·현금 | 보유금액 |

외화 자산은 각 카드 아래에 원화 환산액(`≈ ₩xxx`)이 함께 표시됩니다.

## 4. 거래 내역

### 4-1. 거래 추가

**경로 1**: 자산 상세 페이지 > "거래 추가" 버튼 (해당 자산에 바로 연결)
**경로 2**: 사이드바 > 거래 내역 > "거래 추가" 버튼

### 4-2. 거래 유형별 입력 항목

| 거래 유형 | 설명 | 입력 항목 |
|-----------|------|-----------|
| **매수 (BUY)** | 주식·ETF·가상화폐 매입 | 날짜·수량·단가·수수료·메모 |
| **매도 (SELL)** | 보유 자산 매도 | 날짜·수량·단가·수수료·메모 |
| **입금 (DEPOSIT)** | 예금 불입, 현금 추가 | 날짜·금액·메모 |
| **출금 (WITHDRAW)** | 예금 해지, 현금 인출 | 날짜·금액·메모 |
| **배당/이자 (DIVIDEND)** | 배당금, 이자 수령 | 날짜·금액·메모 |

> **주의**: 매도 수량이 현재 보유 수량을 초과하면 입력이 차단됩니다.

### 4-3. 거래 내역 목록

**경로**: 사이드바 > 거래 내역

전체 거래를 날짜순으로 확인할 수 있습니다.

### 4-4. 거래 삭제

자산 상세 페이지 또는 거래 내역 목록에서 개별 거래를 삭제할 수 있습니다.

## 7. 자산 변동 히스토리

**경로**: 사이드바 > 히스토리

월별 전체 자산 변동 추이를 확인합니다.

### 스냅샷 자동 저장

앱을 실행할 때 **이번 달 스냅샷이 없으면 자동으로 저장**됩니다. 월 1회 실행 기준으로 매월 자산 현황이 기록됩니다.

### 표시 항목

- 월별 총 평가금액 / 총 투자원금 라인 차트
- 스냅샷 목록 (날짜·평가금액·투자원금·유형별 비중)

## 8. 설정

**경로**: 사이드바 > 설정

### 8-1. 환율 설정

USD·JPY에 대한 KRW 기준 환율을 수동 입력합니다.

- 형식: `1 외화 = ? KRW`
- 예: USD = 1350, JPY = 9.0
- 이 값이 모든 외화 자산의 원화 환산에 사용됩니다.
- **자동 갱신 없음** — 환율이 크게 변동할 때마다 수동으로 업데이트하세요.

## 9. 데이터 백업 및 복원

모든 데이터는 브라우저 로컬 저장소(IndexedDB)에 보관됩니다.
**브라우저 데이터 초기화 시 삭제**될 수 있으므로 정기적인 백업을 권장합니다.

### 9-1. JSON 내보내기 (백업)

1. 설정 > 데이터 백업 / 복원 > **"JSON 내보내기"** 버튼 클릭
2. `finance-backup-YYYY-MM-DD.json` 파일이 자동 다운로드됩니다.
3. 자산·거래 내역·스냅샷 전체가 포함됩니다.

### 9-2. JSON 가져오기 (복원)

1. 설정 > 데이터 백업 / 복원 > **"JSON 가져오기"** 버튼 클릭
2. 백업 파일(`.json`)을 선택합니다.
3. 성공 시 페이지가 자동 새로고침됩니다.

> ⚠️ **복원 시 기존 데이터가 모두 교체됩니다.** 현재 데이터가 필요하다면 복원 전에 먼저 내보내기를 실행하세요.

## 10. 자주 묻는 질문

**Q. 현재가를 업데이트하지 않으면 어떻게 되나요?**
현재가가 0인 자산은 목록에서 "현재가 미입력" 배지가 표시되고, 수익률 계산 시 0으로 처리됩니다.

**Q. 예금의 이자는 어떻게 기록하나요?**
거래 추가 > 유형: **배당/이자 (DIVIDEND)** 로 이자 수령액을 기록합니다.

**Q. 부동산을 추가하면 수량은 어떻게 되나요?**
부동산은 자동으로 수량=1로 고정됩니다. "현재 시세" 하나만 입력하면 됩니다.

**Q. 환율은 얼마나 자주 업데이트해야 하나요?**
정확한 원화 환산을 위해 주 1회 또는 환율 변동이 클 때 업데이트하는 것을 권장합니다.

**Q. 여러 기기에서 같은 데이터를 사용하려면?**
한 기기에서 JSON 내보내기 → 다른 기기에서 JSON 가져오기로 동기화할 수 있습니다. 자동 동기화는 지원되지 않습니다(MVP 범위 외).

**Q. 매도 후 수익률이 이상하게 계산됩니다.**
매도 수익률은 **이동평균법**으로 계산됩니다. 매수 단가의 가중 평균을 기준으로 실현손익을 산정합니다.

**Q. 데이터가 사라졌습니다.**
브라우저 설정에서 사이트 데이터를 초기화하면 IndexedDB 데이터도 삭제됩니다. 정기적인 JSON 백업을 권장합니다.
